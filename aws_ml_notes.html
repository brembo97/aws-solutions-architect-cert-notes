Module 1 — Vision & Speech
Rekognition & Transcribe
Computer vision for images and video, plus speech-to-text transcription.
REK
Amazon Rekognition core
Computer vision API — analyze images and videos without ML expertise
▶

Amazon Rekognition is a fully managed computer vision service that analyzes images and videos for objects, scenes, faces, text, and activities using deep learning — no ML knowledge required. You send an image or video to the API and receive structured JSON results.

→ Object and scene detection: identify thousands of objects (cars, trees, food, animals), scenes (beach, city, stadium), and activities (running, dancing). Each detection comes with a confidence score.
→ Facial analysis: detect faces and return attributes — age range, gender, emotions (happy, sad, angry, surprised, etc.), glasses, beard, smile. Also detects face poses and quality metrics.
→ Facial comparison: compare two faces to determine if they belong to the same person. Returns a similarity score. Used for identity verification.
→ Facial search / Celebrity recognition: index a collection of faces and search for matching faces in images or video streams. Also recognizes thousands of public celebrities by name.
→ Content moderation: detect explicit, suggestive, or violent content in images and video. Returns confidence scores — use a threshold to automatically flag or block content. Essential for user-generated content platforms.
→ Text in images (OCR): detect and extract printed and handwritten text from images (different from Textract — Rekognition OCR is simpler, Textract handles structured documents).
→ Video analysis: Rekognition Video processes video asynchronously (stored in S3) or in real-time (via Kinesis Video Streams) — detects activities, tracks people across frames, identifies scenes.
→ Custom Labels: train a custom model on your own labeled images to detect domain-specific objects that Rekognition doesn't detect out-of-the-box (e.g. specific product defects, proprietary logos).
EXAM TIP"Automatically moderate user-uploaded photos" → Rekognition content moderation. "Verify a user's identity by comparing their selfie to their ID photo" → Rekognition facial comparison. "Detect what's in an image" → Rekognition. Rekognition OCR = simple text in images; Textract = structured documents with forms and tables.
TRN
Amazon Transcribe core
Automatic speech recognition — audio/video to accurate text transcripts
▶

Amazon Transcribe converts speech in audio and video files (or live streams) to text using automatic speech recognition (ASR). It is fully managed and serverless — submit audio, receive a transcript.

→ Supports batch transcription (audio/video stored in S3) and real-time streaming transcription (live audio via WebSocket or HTTP/2).
→ Speaker identification (diarization): distinguish and label multiple speakers in a recording — "Speaker 1: ...", "Speaker 2: ...". Useful for call center transcripts, interviews, meetings.
→ Custom vocabulary: improve accuracy for domain-specific terms (medical jargon, product names, acronyms) by providing a custom vocabulary list.
→ PII redaction: automatically identify and redact Personally Identifiable Information (names, addresses, credit card numbers, phone numbers) from transcripts. Critical for compliance.
→ Language identification: automatically detect the language spoken (from a list you specify) without knowing in advance.
→ Supports multiple languages including English, Spanish, French, German, Japanese, Korean, Portuguese, and many more.
EXAM TIP"Convert call center audio recordings to text" → Transcribe. "Transcribe with multiple speakers identified" → Transcribe with speaker diarization. "Remove PII from call transcripts automatically" → Transcribe PII redaction. Transcribe = speech → text. Polly = text → speech (opposite direction).
Module 2 — Speech & Language
Polly, Translate & Lex
Text-to-speech synthesis, language translation, and conversational AI chatbots.
POL
Amazon Polly core
Text-to-speech — lifelike voices in dozens of languages
▶

Amazon Polly converts text into lifelike spoken audio using deep learning. It is the opposite of Transcribe: Transcribe = speech → text, Polly = text → speech.

→ Standard voices: high-quality speech synthesis using traditional concatenative TTS. Neural voices (NTTS): more natural, expressive speech using neural networks. Neural voices sound significantly more human-like but cost more.
→ Newscaster style: a Neural Text-to-Speech style that simulates a professional news anchor voice. Available for select English voices.
→ SSML (Speech Synthesis Markup Language): XML-based markup to control pronunciation, speaking rate, pitch, volume, pauses, emphasis, and even whisper effects.
→ Lexicons: customized pronunciation rules for words and phrases — e.g. define how "AWS" should be pronounced as "Amazon Web Services" or how a brand name should sound.
→ Output formats: MP3, OGG, PCM. Can stream audio in real-time or store to S3.
→ Use cases: accessibility (read articles aloud), e-learning (narrated courses), IVR systems (phone menu prompts), dynamic notifications (read news/reports), multilingual apps.
EXAM TIPPolly = text → speech (Transcribe is the inverse). "Build a screen reader for visually impaired users" → Polly. "Dynamically generate audio notifications" → Polly. Neural voices = more natural, more expensive. SSML = fine-grained pronunciation and style control.
TRL
Amazon Translate core
Neural machine translation — accurate, fluent translation between 75+ languages
▶

Amazon Translate is a neural machine translation service that delivers fast, high-quality, and customizable language translation. It can translate text in real-time or in batch at scale.

→ Supports 75+ languages. Translate between any supported language pair — not just to/from English.
→ Custom terminology: define domain-specific terms that must be translated in a specific way (brand names, technical terms, product names) — overrides the default translation for those terms.
→ Active Custom Translation (ACT): fine-tune the translation model on parallel texts (source + reference translations) specific to your domain for higher quality.
→ Use cases: localize websites/apps for international markets, translate user-generated content (reviews, support tickets), real-time chat translation, multilingual document processing.
EXAM TIP"Translate user reviews into English from multiple languages" → Amazon Translate. "Translate website content dynamically for international users" → Amazon Translate. One-liner: Translate = language conversion at scale. Don't confuse with Comprehend (which detects language and analyzes sentiment, but doesn't translate).
LEX
Amazon Lex + Connect core
Conversational AI chatbot engine — the technology behind Alexa, integrated with Amazon Connect
▶

Amazon Lex is a fully managed service for building conversational interfaces using voice and text. It uses the same ASR and NLU technology that powers Amazon Alexa. Amazon Connect is AWS's cloud-based contact center service built on top of Lex.

→ Lex capabilities: ASR (convert speech to text input), NLU (understand intent from text), manages multi-turn conversations (remembers context across turns), slots (gather specific pieces of information from users).
→ Intents: what the user wants to do (e.g. "BookFlight", "CheckBalance"). Slots: pieces of information needed to fulfill the intent (departure city, date, seat class). Lex prompts for missing slots.
→ Lambda integration: Lex triggers a Lambda function to fulfill the intent (look up data, make a booking, call an API). Lambda handles the business logic.
→ Amazon Connect: cloud contact center service. Handles inbound/outbound calls with IVR flows powered by Lex. Can route calls to agents, play prompts, gather customer input via voice — all serverless. No per-seat licensing.
→ Typical flow: Customer calls → Connect IVR → Lex understands intent → Lambda fulfills → response played back via Polly (TTS).
EXAM TIP"Build a chatbot that understands natural language" → Amazon Lex. "Build a call center IVR with AI understanding" → Amazon Connect + Lex. "The technology behind Alexa" = Lex. Lex handles conversation management; Lambda handles business logic. Connect = the contact center layer on top of Lex.
Module 3 — Language & Documents
Comprehend & Textract
Natural language processing for text analysis, and intelligent document extraction from PDFs and scanned forms.
COM
Amazon Comprehend core
NLP — extract meaning, sentiment, entities, and topics from text
▶

Amazon Comprehend is a fully managed NLP (Natural Language Processing) service that extracts insights from unstructured text — customer reviews, social media posts, emails, support tickets, documents — without requiring ML expertise.

→ Sentiment analysis: determine if text is Positive, Negative, Neutral, or Mixed. Returns confidence scores for each. Use for analyzing product reviews, social media, support tickets, or surveys.
→ Entity recognition (NER): identify and classify named entities in text — People, Organizations, Locations, Dates, Quantities, Events, Products, Titles. Returns the entity text and its type.
→ Key phrase extraction: identify the most important phrases in a document that describe the main topics (without needing full entity classification).
→ Language detection: automatically identify which language a piece of text is written in from 100+ supported languages. Returns confidence score per detected language.
→ PII detection and redaction: detect and optionally redact Personally Identifiable Information in text — names, addresses, SSNs, credit card numbers, phone numbers.
→ Topic modeling: discover latent topics across a collection of documents using Latent Dirichlet Allocation (LDA). Groups documents by theme without predefined categories.
→ Custom Classification and Custom Entity Recognition: train custom NLP models on your own labeled data for domain-specific categories (e.g. classify support tickets into your own categories, or recognize proprietary entity types).
EXAM TIP"Analyze sentiment of customer reviews" → Comprehend. "Extract named entities from news articles" → Comprehend. "Detect the language of incoming messages" → Comprehend. "Classify support tickets into custom categories" → Comprehend Custom Classification. Comprehend = analyze/understand text. Translate = convert text to another language. Transcribe = audio to text.
MED
Amazon Comprehend Medical core
Clinical NLP — extract medical entities from unstructured clinical text
▶

Comprehend Medical is a specialized version of Comprehend trained on medical and clinical language. It extracts medical information from unstructured clinical text — doctor's notes, discharge summaries, lab reports, prescriptions.

→ Medical entity extraction: identifies medical entities — Medications (drug name, dosage, route, frequency), Medical Conditions (diagnoses, signs, symptoms), Anatomy (body parts, systems), Test/Treatment/Procedure (lab tests, imaging, treatments), Time expressions.
→ Protected Health Information (PHI) detection: identify and locate PHI in clinical text for HIPAA compliance — patient names, dates, addresses, identifiers.
→ ICD-10-CM and RxNorm ontology linking: map detected conditions to ICD-10-CM codes (standard diagnosis codes) and medications to RxNorm concepts (standard drug database).
→ Use cases: clinical decision support, pharmacovigilance (adverse event detection), patient timeline extraction, clinical trial matching, revenue cycle automation.
EXAM TIP"Extract medical information from clinical notes" → Comprehend Medical. "Detect PHI in patient records for HIPAA compliance" → Comprehend Medical. "Map diagnoses to ICD-10 codes" → Comprehend Medical. It is Comprehend specialized for clinical/medical language — not general-purpose NLP.
TXT
Amazon Textract core
Intelligent document extraction — beyond OCR, understands forms and tables
▶

Amazon Textract extracts text and structured data from scanned documents, PDFs, and images. Unlike basic OCR tools, Textract understands document structure — it can extract form fields (key-value pairs), table data (rows and columns), and preserve the semantic layout of the document.

→ Text extraction: detects and extracts all printed and handwritten text from documents, preserving spatial positioning.
→ Form extraction: identifies key-value pairs in forms — e.g. "Name: John Smith", "Date of Birth: 01/01/1990". Understands that a label and its adjacent value belong together.
→ Table extraction: detects tables in documents and extracts cell content with row/column coordinates — preserving the structure of financial statements, invoices, and data tables.
→ Queries API: ask specific questions about a document (e.g. "What is the patient's date of birth?" or "What is the invoice total?") and Textract returns the answer — without processing the entire document structure.
→ Specialty document analysis: pre-built models for Expense documents (invoices, receipts), ID documents (passports, driver's licenses), Lending documents (W-2, 1099, mortgage applications).
→ Handles both synchronous (real-time, single page) and asynchronous (multi-page PDFs stored in S3) processing.
→ Use cases: automating invoice processing, digitizing medical forms, extracting data from tax documents, processing insurance claims, onboarding document verification.
EXAM TIPTextract vs Rekognition OCR: Rekognition OCR = detect text in images (simple). Textract = extract structured data from documents (forms, tables, key-value pairs). "Extract form fields from scanned PDFs" → Textract. "Process invoices and receipts automatically" → Textract Expense Analysis. Textract understands document layout and structure; Rekognition just detects text characters.
Module 4 — Platform & Search
SageMaker AI & Kendra
Build, train and deploy custom ML models with SageMaker; add intelligent enterprise search with Kendra.
SGM
Amazon SageMaker AI core
End-to-end ML platform — build, train, tune, and deploy custom models at scale
▶

Amazon SageMaker is a fully managed platform for the complete machine learning lifecycle — from data preparation and model training to deployment and monitoring. Unlike the other AWS ML services (which are pre-built APIs), SageMaker is for building your own custom models when the pre-built services don't fit your use case.

→ SageMaker Studio: web-based integrated development environment (IDE) for ML — notebooks, training jobs, experiments, model registry, and deployment all in one UI.
→ SageMaker Notebooks: fully managed Jupyter notebooks with on-demand compute. No server setup — spin up a GPU-backed notebook in seconds.
→ Training Jobs: run training on managed infrastructure. SageMaker provisions the compute (any instance type), runs your training script, saves the model artifact to S3, then terminates the instance. Pay only for training time.
→ Automatic Model Tuning (HPO): automatically find the best hyperparameters for your model using Bayesian optimization — runs many training jobs in parallel and tracks which parameters produce the best results.
→ SageMaker Endpoints: deploy trained models as real-time HTTPS inference endpoints. Fully managed — auto-scaling, A/B testing between model versions, multi-model endpoints (host multiple models on one endpoint to reduce costs).
→ Batch Transform: run inference on large datasets stored in S3 without a live endpoint. Cost-effective for bulk predictions that don't need real-time latency.
→ SageMaker Pipelines: CI/CD for ML — build repeatable, automated ML workflows (data processing → training → evaluation → deployment) triggered by code commits or schedules.
EXAM TIPSageMaker is the answer when existing AI services (Rekognition, Comprehend, etc.) don't solve the problem and a custom model is needed. "Train a custom ML model" → SageMaker. "Deploy a trained model as an API endpoint" → SageMaker Endpoint. For the exam, SageMaker is often the answer to "none of the pre-built AI services fit this specific use case."
KEN
Amazon Kendra advanced
Intelligent enterprise search — natural language search across internal documents
▶

Amazon Kendra is a fully managed intelligent search service powered by machine learning. It enables natural language search across your organization's internal documents and data sources — returning direct answers, not just links to documents.

→ Unlike traditional keyword search, Kendra understands natural language queries — "What is the vacation policy?" returns the specific paragraph, not a list of documents containing those words.
→ Data source connectors: index content from S3, SharePoint, Confluence, Salesforce, ServiceNow, RDS, OneDrive, Slack, GitHub, Jira, and many more — without building custom ingestion pipelines.
→ Supports document formats: PDF, Word, HTML, PowerPoint, and plain text.
→ Incremental learning: learns from user interactions (which results users click) to continuously improve search relevance over time.
→ Use cases: internal knowledge base search, HR policy queries, IT helpdesk automation, customer support knowledge search, regulatory document search.
EXAM TIP"Employees search internal company documents using natural language questions" → Amazon Kendra. Kendra = intelligent enterprise search (understands intent, returns direct answers). Contrast with OpenSearch (Elasticsearch-compatible, keyword/full-text search, analytics). Kendra is ML-powered; OpenSearch is traditional search.
Module 5 — Personalization
Personalize & Connect
Real-time ML-powered recommendations for every user, and the full Amazon Connect contact center picture.
PER
Amazon Personalize core
Real-time personalized recommendations — same ML technology as Amazon.com
▶

Amazon Personalize is a fully managed ML service for building real-time personalized recommendations — product recommendations, content suggestions, search ranking, and user segmentation. It uses the same recommendation technology AWS built for Amazon.com.

→ Takes in three types of data: User data (demographics, attributes), Item data (product catalog, content metadata), and Interaction data (clicks, purchases, views, ratings — the most important signal).
→ Automatically selects, trains, and tunes the right ML algorithm based on your data and use case — no ML expertise needed.
→ Use cases: "Customers who bought this also bought…", "Recommended for you", trending items, similar items, personalized search ranking, re-engagement campaigns (which items to promote to a specific user via email/push).
→ Real-time recommendations: deployed as an API — call the endpoint with a user ID and get back ranked recommendations in milliseconds. Also supports batch recommendations for offline use (email campaigns).
→ Data stays entirely in your AWS account — Personalize trains private models on your data, not shared models. Results are specific to your business.
EXAM TIP"Add product recommendations to an e-commerce site" → Amazon Personalize. "Personalize content for each user on a media streaming platform" → Amazon Personalize. The key differentiator: Personalize trains ML models on YOUR interaction data. SageMaker can also build recommendation systems but requires ML expertise to build from scratch.
CON
Amazon Connect — Full Picture core
Cloud contact center — omnichannel, AI-powered, no hardware required
▶

Amazon Connect is AWS's cloud-native contact center service — an omnichannel platform for handling customer calls, chat, and tasks. It integrates with Lex (conversational AI), Polly (TTS), Lambda (business logic), and other AWS services to create intelligent, serverless contact center flows.

→ No hardware or telephony infrastructure required — provision a contact center in minutes. Pay per minute of call + per message for chat. No agent seat licenses.
→ Contact flows: visual drag-and-drop builder for IVR flows — play prompts (Polly), gather DTMF/speech input (Lex), route to agents or queues, trigger Lambda for lookups, set attributes based on conditions.
→ Full architecture for an AI-powered call center: Caller dials → Connect receives → Lex handles NLU → Lambda does CRM lookup → Polly reads back response → route to human agent if needed.
→ Integrates with CRMs (Salesforce, Zendesk), WFM tools, and AWS analytics (Kinesis → QuickSight dashboards for real-time contact center metrics).
EXAM TIP"Build an AI-powered call center with no servers" → Amazon Connect + Lex + Polly + Lambda. Amazon Connect is the exam's standard answer for "contact center" or "IVR" scenarios. Often appears combined with Lex (NLU), Polly (TTS), and Lambda (business logic) in a single architecture question.

CMP
Easy-to-Confuse Pairs core
The most common exam distractors — know exactly how these services differ
▶
PairHow to tell them apart
Transcribe vs PollyTranscribe = speech → text. Polly = text → speech. Exact opposites. "Transcribe converts audio to text; Polly converts text to audio."
Rekognition OCR vs TextractRekognition OCR = detect text characters in an image (simple). Textract = extract structured data from documents — forms, tables, key-value pairs (intelligent). Use Textract for PDFs/documents; Rekognition for "what text appears in this photo."
Comprehend vs TranslateComprehend = analyze text (sentiment, entities, language detection, topic modeling). Translate = convert text to another language. Comprehend understands; Translate converts.
Kendra vs OpenSearchKendra = ML-powered natural language enterprise search (asks a question, gets a direct answer). OpenSearch = Elasticsearch-based full-text search (keyword matching, log analytics). Kendra understands intent; OpenSearch matches keywords.
Personalize vs RekognitionPersonalize = recommend items/content to users based on behavior. Rekognition = analyze what's in images/video. Completely different domains.
Comprehend vs Comprehend MedicalComprehend = general-purpose NLP on any text. Comprehend Medical = specialized for clinical/medical language (trained on medical text, maps to ICD-10/RxNorm, detects PHI for HIPAA).
SageMaker vs pre-built AI servicesPre-built services (Rekognition, Comprehend, etc.) = ready-to-use APIs, no ML expertise needed. SageMaker = build and train your own custom model when pre-built services don't solve your specific problem.
Lex vs ConnectLex = the NLU/chatbot engine (understands intent from voice/text). Connect = the contact center platform that uses Lex as its AI brain. Connect routes calls; Lex understands them.
